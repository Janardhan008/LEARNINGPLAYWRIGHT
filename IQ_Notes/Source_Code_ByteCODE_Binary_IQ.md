# Source Code vs Bytecode vs Binary Code

## Example

```js
console.log("Hello, Jan!");
```

---

## Comparison Table

| Concept      | Description | How the Example Looks |
|--------------|-------------|----------------------|
| **Source Code** | Human-readable instructions written by a programmer in a high-level language (JS, Python, Java, etc.). It uses syntax that humans can understand and edit. | `console.log("Hello, Jan!");` — plain text, readable English-like syntax. |
| **Bytecode** | Intermediate, platform-independent representation generated by a compiler or interpreter (e.g., V8 for JS, JVM for Java). The interpreter/virtual machine executes this. It is not meant for humans to read. | `0x2e, 0x00, 0x01, 0x02, LdaUndefined, CallRuntime` — instruction mnemonics and numeric opcodes (e.g., V8 Ignition bytecode). |
| **Binary Code (Machine Code)** | The lowest-level representation — raw bits (0s and 1s) that the CPU fetches and executes directly. It is platform-specific (x86, ARM, etc.) and completely unreadable to humans. | `10110000 01100001 11000011 10100010 00000000 00010111 00000000 00000000` — raw binary or hex that the CPU decodes as instructions. |

---

## How They Relate

```
Source Code  ──(parse/compile)──▶  Bytecode  ──(JIT/interpret)──▶  Binary Code
  (text)                               (intermediate)                  (CPU runs this)
```

1. You write **source code** (`console.log("Hello, Jan!");`).
2. The JavaScript engine (V8) parses and compiles it into **bytecode** (Ignition bytecode).
3. The engine may further JIT-compile hot bytecode into **binary machine code** for fast execution.
